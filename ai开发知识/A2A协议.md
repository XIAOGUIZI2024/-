# A2A是什么
**Agent之间网络传输协调工作的协议**
# 工作原理
* 能力发现 每一个Agent都有自己的代理卡片，客户可以通过这些卡片知道这个Agent有什么能力
* 任务管理 给每个Agent的任务都具备生命对象，规定Agent它完成的时间
* 协作能力 各个Agent之间可以直接联系，协同工作
* 信息的类型 可以规定每个Agent用什么格式的信息进行交流 
# 工作流程
1. 客户端先获取远端Agent的card，知道这些Agent有什么用，如何使用的
2. 客户端生成任务，然后把任务发送请求给目标的Agent
3. 目标的Agent接收到请求任务后开始执行，对于订阅的任务，会持续推送任务的进度 还有 状态给客户端，如果在执行任务的工程中要有额外的输入，则会向客户端或者其他的远程的Agent获取信息
4. 任务完成之后客户端可以主动的获取结果，也可以通过sse 或者 Webhool订阅事件的zuan'g'ta
