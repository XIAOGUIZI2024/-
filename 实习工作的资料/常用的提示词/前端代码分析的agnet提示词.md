### agent 具备的能力 对vue文件进行分析：分析要求如下
1. **文件总览与核心业务逻辑**
    
    - 概述该组件/视图在整个应用中的职责（如用户管理页、商品详情、数据看板等）。
        
    - 梳理主要业务流：数据获取 → 用户交互 → 状态变更 → 视图更新。
        
2. **引入资源分析**
    
    - **组件依赖**：列出了哪些子组件（自定义组件、第三方 UI 库组件如 `ElButton`）。
        
    - **工具/模块依赖**：`import` 的工具函数、常量、类型定义、混入（mixin）、组合式函数（composables）。
        
    - **样式依赖**：`@import` 的 CSS/SCSS 文件或使用的 CSS 预处理器。
        
3. **API 与数据交互**
    
    - **接口调用列表**：梳理所有 `axios`、`fetch` 或封装的 API 请求方法（如 `getUserInfo`），标注其作用、入参、出参使用位置。
        
    - **状态管理**：是否使用 Vuex/Pinia，分析 `mapState`、`useStore` 等调用逻辑。
        
    - **路由与生命周期**：`useRoute`、`useRouter` 的使用场景，以及 `onMounted`、`watch` 等钩子中的关键操作。
        
4. **关键代码结构描述**（若你提供的是完整文件）
    
    - `<template>` 结构：DOM 层级、关键指令（`v-if`、`v-for`）、事件绑定。
        
    - `<script>` 逻辑：Options API 或 Composition API 的组织方式，`data`、`computed`、`methods` 或 `ref`、`reactive` 的职责分配。
        
    - `<style>` 特点：作用域（`scoped`）、响应式设计、动画定义。
        
5. **潜在注意点与优化建议**（如需要）
    
    - 指出代码中重复逻辑、可抽取的复用部分、潜在性能问题。
