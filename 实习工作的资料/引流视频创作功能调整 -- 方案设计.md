# 新闻体视频 -- 方案设计
## 具体的修改细节
## 1. 功能拆解

### ✅ 核心主流程（用户视角）

**流程一：创作 → 生成记录**

1. 用户进入「视频创作详情页」
2. 在五大区域完成配置：
    - 吸睛封面区
    - 黄金三秒区
    - 混剪素材区
    - 精彩收尾区
    - 背景音乐区
3. 点击「开始创作 / 批量生产」
4. 系统生成一个“创作任务”
5. 跳转/或提示用户在「创作记录列表」查看状态

---

**流程二：查看创作记录**

1. 用户进入「创作记录列表页」
2. 查看历史任务（含批量任务）
3. 每条记录展示：
    - 任务ID
    - 创建时间
    - 批量数量
    - 状态（制作中 / 成功 / 失败）

---

**流程三：进入详情页（复用创作数据）**

1. 用户点击记录
2. 若状态 = 成功 / 失败 → 允许进入
3. 跳转到「创作详情页」
4. 页面加载当时的创作数据（五大区域）

---

**流程四：修改并自动保存**

1. 用户在五大区域修改内容
2. 系统自动调用保存接口（分区保存）
3. 数据实时更新

---

**流程五：素材库使用**

1. 用户进入素材库页面：
    - 上传素材
    - 搜索素材
    - 删除/修改
2. 在创作详情页选择素材进行使用

---

### ✅ 子功能模块拆解

#### 1. 创作记录模块

- 记录任务（批量）
- 状态管理（制作中 / 成功 / 失败）
- 记录数量展示
- 点击跳转控制（状态限制）

#### 2. 创作详情回填模块

- 历史数据恢复
- 区域级数据加载

#### 3. 自动保存模块（重点）

- 分区保存（type字段）
- 仅“从记录进入”才触发

#### 4. 批量生产模块（已有，需兼容）

- 一个任务 → 多个结果
- 记录层 vs 实际视频层的关系

#### 5. 素材库模块

- 用户私有素材
- 上传 / 删除 / 修改
- 搜索 + 分页
- 创作页面引用

---

### ⚠️ 边界条件 & 异常

1. **任务未完成点击**
    - 状态=制作中 → 禁止进入
2. **批量任务但数据一致**
    - 一个任务ID → 多个视频
    - 但创作配置只有一份（关键点）
3. **自动保存误触**
    - 新建创作不能自动保存（必须区分来源）
4. **并发保存冲突**
    - 多区域同时修改
5. **素材删除后引用问题**
    - 创作中素材被删除怎么办？
## 数据库的建表设计

#### 创作记录表（creation_record）

| 字段               | 类型       | 说明                 |
| ---------------- | -------- | ------------------ |
| id               | int      | 主键                 |
| user_id          | bigint   | 用户                 |
| task_id          | varchar  | 任务ID               |
| task_name        | varcher  | 任务的名称              |
| status           | tinyint  | 状态 0-制作中，1-成功，2-失败 |
| created_at       | datetime | 创建时间               |
| updated_at       | datetime | 更新时间               |
| result_video_url | varchar  | 生成后的视频地址           |
| is_delete        | tinyint  | 逻辑删除 - 0存在 1 -删除   |
#### 创作内容表（creation_content）

| 字段         | 类型       | 说明               |
| ---------- | -------- | ---------------- |
| id         | bigint   | id               |
| task_id    | varchar  | 关联任务的id          |
| type       | varchar  | 编辑的类型            |
| content    | string   | 编辑的类型            |
| created_at | datetime | 创建时间             |
| updated_at | datetime | 更新时间             |
| is_delete  | tinyint  | 逻辑删除 - 0存在 1 -删除 |

👉 核心设计点：**按type拆分**
