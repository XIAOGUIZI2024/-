# 实现人物建模与专属声音关联绑定
#### 按照目前的业务流程来说，由于小冰与闪剪的流程是不一样的，所以人物建模与专属声音的关联绑定这里没有很明显的流程顺序，我们无法在用户体验极好的情况下进行这两个的强关联，按照强行关联的要求，实现这个功能的前提是要给用户一个进行关联的步骤，不然的话，从技术层面上我们无法进行绑定的功能，但是这个就会导致用户的体验非常的差劲。
#### 经过我们团队的商量，我们决定从业务上面进行优化的实现，这个功能的添加主要是为了让建模与它相关的人物相关的声音进行绑定，这样的话我们为何不直接拿建模人物的名称对专属的声音进行搜索呢
#### 我们试想一个具体的业务场景 将设有 克隆人AXX ，克隆人AXY 这个两个,他们的声音有 声音AXX,声音AXY,声音AYY。这个可以看出来这人物A就是想要进行关联的，所以我们主要在用户进行声音的命名 或者 克隆人的命名 这两个步骤进行友好的提示，就可以完美的解决了
```
1.在【选择声音建模】展示框里面的头部添加 {两个而选择一的}【按钮】-- [详细] 与[全部] 
2. 在【开始创作】Tab下的【选择数字人】展示框中实现点击具体的数据人信息，接着在【选择声音建模】展示框中点击[详细]按钮就会触发搜索克隆人具体的声音，也就是把这个克隆人关联的专属声音进行展示，但是z注意的是在[全部]按钮下，试是要展示所以的声音的
```
