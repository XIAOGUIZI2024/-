# MYSQL有多少种存储引擎
* InnoDB
  这个是mysql5.5版本以后默认的存储引擎，其支持事务，行级锁，mvcc，外键，聚簇索引，b+树索引，全文索引 等适合高比发的OLTP场景
* MYISAM
  MYSQY老版本默认的存储引擎，只用表级锁，不支持事务，但是读数据的能力很强，适合读多写少的场景
* MEMORY
  这个索引的数据都是存储在内存上的，读写数据的能力极强，但是mysql重启后的数据就没有了
* Archive
  这个不支持索引，专门用来做归档数据
* NDS
  这个支持分布式和高可用，数据自动分片，适合电信级别的大规模集成
# InnoDB 和 MyISAM的差别
* 从存储结构上看 
  1. MyISAM它的索引和数据的存储是分开的，.MYI文件存储的是索引，而.MYD文件存储的是数据
  2. InnDB 数据直接挂在B+树的叶子节点上
* 锁机制的差别
  1. MyISAM只可以锁住一张表，并发起来性能很差
  2. InnoDB支持行锁，通过MVCC实现读写不阻塞
