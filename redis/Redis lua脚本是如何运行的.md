1. **Lua 脚本定义**
    
    Lua 是轻量脚本语言，Redis 内置 Lua 解释器；Redis Lua 脚本可将多条 Redis 命令合并，在服务端一次性执行，具备**原子性、减少网络开销、支持条件逻辑**的特点。
    
2. **Redis 执行 Lua 两大核心命令**
    

- `EVAL`：直接传入完整 Lua 脚本即时执行，适合测试，缺点是每次传输完整脚本、重复编译、性能差。
- `EVALSHA`：先通过`SCRIPT LOAD`预加载脚本并缓存，生成唯一 SHA1 值；后续通过短 SHA 值执行脚本，**节省带宽、只编译一次、性能更高**，是生产环境首选。

3. **Lua 脚本执行过久的危害**
    
    Redis 为**单线程**模型，Lua 脚本执行期间独占线程：

- 所有其他客户端命令全部阻塞，业务请求超时、服务卡顿甚至雪崩；
- 触发默认 5 秒超时限制。

4. **超时脚本两种场景 & 处理**

- 场景一：脚本**未修改任何数据**：可执行 `SCRIPT KILL` 直接终止脚本，恢复 Redis 正常。
- 场景二：脚本**已修改数据**：为保证原子性、避免数据不一致，**无法手动杀死脚本**，只能等待脚本执行完毕；若为死循环卡死，最终只能**重启 Redis**应急。

5. **核心使用规范**
    
    Lua 脚本必须极简、轻量、快速执行；禁止写死循环、大量循环、复杂计算；通过合理配置超时时间、规范脚本编写，提前规避阻塞风险。
