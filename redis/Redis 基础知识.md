# 常见的数据类型
* String：这个是最常见的数据类型，能够存储数字，字符串，二进制数，最大的存储内存是512MB，主要是用做缓存，计数器，分布式锁
* List：一个有序的字符串列表，底层是双向链表，支持两端的操作，主要是用作在消息队列，时间轴，最新的信息列表
* Set：无序不可以重复的集合，主要是用作去重
* Hash：一个建值对集合存储结构化数据（对象的信息）
* Zset：带有一个score分数进行排序的set，主要是用作排行榜
* BitMap：用位来存储数据的，每一个bit只占0或者1，空间的使用程度很高，主要是用在用户的签到，在线状态统计
* Stream：5.0版本引入的，专门为消息队列设计的，两个关键的特点，其一会自动生成全局唯一的信息ID，其二可以信息持久化，主要是用于消息队列
* GEO：用来存储地理的位置信息，支持经纬对和空间存储与查询，主要是附件的人/店铺，地理位置的标记
* HyperLogLog：概率算法，用极小内存（固定12KB）统计巨大数据集的基数（去重计数），是一个概率性数据结构，专门用来估算基数，主要是用于搜索关键词去重计算
# Hash基础知识
* 数据量小时可以才用listpack来存储，这个其实就是一个链表，数据依次存储在链表上，查询的时间复制度是O（n）,没有指针的开销
* 数据量大的时候才有Hashtable来存储数据，这个是一个数组+ 链表的结构，内存是分散的，有只指针的开销
* 转变的条件，如果某个字段超过512，或者任意的一个字段值超过64字节，他就会从listpack h转换为Hashtable的扩容
* 渐进式 rehash，这个主要是解决hash扩容不是一次性搬完的，而是多次慢慢搬完的，先给ht[1]分配空间，大小是大于ht[0].used 2 的2次幂，接着把rehashidx 的值从-1改为0 ，每次Hash做增删改查的操作，rehashidx对应的桶都是从ht[0]般到ht[1]中，然后rehashidx都会进行++的操作，插入的新数据都是存储在ht[1]上，但是查询的数据两个表都要查询，搬完之后的ht[0] 与 ht[1]指针进行互换，rehashidx 重置为 -1
# ZSet基础知识
* 底层是 跳表 + 哈希表双结构配合
   - **按成员操作**：比如 `ZSCORE` 获取某个成员的分数，`ZREM` 删除某个成员。这些操作要求能通过成员名快速定位到其分数，类似哈希表 O(1) 的查找性能。
   - **按分数范围操作**：比如 `ZRANGE` 按分数范围获取成员列表，`ZRANK` 获取某个成员的排名。这些操作要求数据按分数有序存储，并且能高效地进行范围遍历，类似跳跃表的特性。
单独使用任何一种结构都无法高效满足所有需求：
  -  只用哈希表：无法按分数排序，范围查询会非常低效（需要全局排序）。
    
  - 只用跳表：按成员查找分数需要 O(log N) 的跳表查找，虽然不慢，但不如哈希表 O(1) 快；而且跳表维护的是成员和分数的映射，按成员删除/更新时仍需先找到该节点。
* 运用的场景
1. 排行榜
2. 延迟对列
3. 滑动窗口限流
