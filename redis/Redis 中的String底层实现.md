## 1. String 底层核心载体

Redis String **不使用 C 原生字符串**，底层依靠 **SDS（简单动态字符串）** 实现。

## 2. SDS 四大核心优势（对应 C 语言字符串缺陷）

1. **O (1) 获取长度**：自带 len 字段，不用遍历；
2. **二进制安全**：依靠长度标识数据，不会被内容里的 `\0` 截断，可存图片、字节流；
3. **杜绝缓冲区溢出**：修改数据前自动校验空间，不足则自动扩容；
4. **空间优化机制**
    
    - 空间预分配：扩容时多预留空间，减少频繁内存申请；
    - 惰性释放：字符串缩短时，只修改长度、不立即释放多余内存，方便复用。
    

## 3. redisObject 统一包装层

1. 含义：Redis 所有数据（String、List、Hash 等）的**统一外层包装壳**；
2. 核心字段：
    
    - type：数据类型（String）
    - encoding：底层编码方式
    - ptr：指向真实数据
    
3. 为什么要包装：
    
    - 屏蔽底层不同结构的差异，统一管理；
    - 实现编码自动切换、内存优化、过期淘汰、LRU、对象共享；
    - 上层代码不用关心底层复杂实现。
    

## 4. String 三种底层编码（encoding）

1. **INT**：存储整型数字，无 SDS，最省内存、性能最高；
2. **EMBSTR**：≤44 字节短字符串，redisObject 和 SDS 内存连续，只读高效；
3. **RAW**：长字符串 / 修改后的 EMBSTR，独立 SDS 结构，支持动态扩容。

## 5. 整体逻辑串联

Redis String

→ 外层：`redisObject`（统一管理 + 标记编码）

→ 内层：根据内容自动选择 INT / EMBSTR / RAW

→ 真实存储：依靠 SDS 结构，解决 C 字符串所有缺点，保证高性能、二进制安全。
