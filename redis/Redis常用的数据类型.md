## 一、基础 5 种

### 1. String 字符串

- 特点：简单键值对，可存文本 / 数字 / 二进制，支持原子自增
- 应用：通用缓存、计数器、分布式锁、验证码
- 常用命令：`SET`、`GET`、`INCR`、`DECR`、`EXPIRE`
- 示例：`INCR article:read:10086` 文章阅读量自增

### 2. Hash 哈希

- 特点：一个 key 对应多个 field-value，适合结构化数据
- 应用：存储用户信息、商品数据，局部更新，节省内存
- 常用命令：`HSET`、`HGET`、`HGETALL`、`HINCRBY`
- 示例：`HSET user:1001 name 李四 age 22`

### 3. List 列表

- 特点：**有序、可重复**，链表结构，两头操作
- 应用：消息队列、任务队列、朋友圈时间线、动态列表
- 常用命令：`LPUSH`、`RPUSH`、`LPOP`、`LRANGE`

### 4. Set 集合

- 特点：**无序、自动去重**，支持集合交集 / 并集运算
- 应用：签到去重、共同好友、黑名单、随机抽奖
- 常用命令：`SADD`、`SMEMBERS`、`SINTER`

### 5. ZSet 有序集合

- 特点：**去重 + 自动排序**，绑定 score 分数排序
- 应用：热搜排行榜、积分排名、延时任务、权重队列
- 常用命令：`ZADD`、`ZREVRANGE`、`ZRANK`

## 二、高级 3 种

### 6. Bitmap 位图

- 特点：基于 String 实现，**按二进制位存储**，极度省内存
- 应用：用户每日签到、在线状态判断、海量状态标记
- 常用命令：`SETBIT`、`GETBIT`、`BITCOUNT`

### 7. HyperLogLog

- 特点：**基数统计**，占用内存极小，允许轻微误差，不存原始数据
- 应用：统计页面 UV、独立访客、海量数据去重计数
- 常用命令：`PFADD`、`PFCOUNT`

### 8. GEO 地理位置

- 特点：存储经纬度坐标，支持距离计算、范围查找
- 应用：附近的人、附近门店、配送距离定位
- 常用命令：`GEOADD`、`GEORADIUS`

## 极简背诵版

1. String：缓存、计数器、分布式锁
2. Hash：存储对象、用户 / 商品信息
3. List：队列、时间线、有序可重复
4. Set：去重、共同好友、无序唯一
5. ZSet：排序、排行榜、有序唯一
6. Bitmap：签到、二进制位、省内存
7. HyperLogLog：UV 统计、海量基数计算
8. GEO：经纬度、附近地点
