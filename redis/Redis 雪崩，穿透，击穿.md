- 缓存穿透：**请求数据缓存、数据库均不存在**，非法请求直击数据库；
- 缓存击穿：**热点数据存在，缓存恰好过期**，高并发瞬间压垮数据库；
- 缓存雪崩：分两种情况，一是大量 Key 集中过期，二是**Redis 整体宕机 / 集群故障**，全量流量涌入数据库。

3. 关键深度辨析
    
    你提出关键疑问：**热点 Key 永不过期 + 定时异步刷新** 仅能解决「Key 集中过期」引发的雪崩；**无法应对 Redis 宕机**的雪崩场景，该方案存在适用边界，不能一概而论。
4. 完整兜底方案补充
    
    Redis 宕机类雪崩，需依靠：Redis 高可用架构（主从 / 哨兵 / 集群）、**本地多级缓存**、限流 + 熔断 + 服务降级组合实现防护。
5. 收获：理清三种缓存问题的本质区别，同时区分不同雪崩场景的差异化解决方案，避免方案混用、误区理解。
