1. **两个核心工作**
    
    Redis 处理一次请求只有两步：

- 网络 IO：收包、拆包、发包（**慢、容易堵车**）
- 数据操作：增删改查内存数据（**极快、一瞬间**）

2. **Redis 6.0 分工模式**

- **多线程**：只负责**网络 IO**
    
    并发收客户端请求、并发返回结果，解决**网络 IO 瓶颈**（路没问题，接单的人少才堵）
- **单主线程**：只负责**执行命令、读写内存数据**

3. **你最关心的问题答案**
    
    多线程接收一堆请求后，会整理好排队交给主线程；
    
    因为**内存操作速度远超网络速度**，单线程完全吃得完所有请求，不会堵塞、不会过载。
    
4. **为什么不改成全多线程处理数据？**
    
    全程单线程改数据，**不用加锁、没有并发冲突**，数据绝对安全、代码简单、性能损耗低。
    
5. **核心一句话**
    
    **慢的网络用多线程分摊，快的数据操作保持单线程；既提升并发吞吐，又保证安全无锁**
