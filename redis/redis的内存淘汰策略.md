1. **Redis 8 种内存淘汰策略**
    
    分为两大类：

- `allkeys-xxx`：全局所有 key 都参与淘汰
- `volatile-xxx`：**只淘汰设置了过期时间的 key**
    
    常用三种：`allkeys-lru`、`allkeys-lfu`、`volatile-lru/volatile-lfu`；随机、ttl、noeviction 线上基本不用。

2. **策略生效范围**
    
    淘汰策略、最大内存 **全局 Redis 实例生效**，0~15 号数据库只是逻辑隔离，**不分库单独限制、单独淘汰**，多库数据共用内存、混合淘汰。
    
3. **你的业务最优方案**
    
    选用 **volatile 系列策略（volatile-lru /volatile-lfu）**：
    

- 核心业务 key**不设置过期时间**，永远不会被内存淘汰删除；
- 临时缓存、验证码、Token 等**设置过期时间**，内存溢出时只会清理这类数据；
- 完美实现：核心数据不丢，临时缓存自动回收。

4. **LRU 与 LFU 区别**

- LRU：只看**最近是否使用**，适合短期、波动型热点；
- LFU：看**访问频率 + 定时衰减**，长期固定热点更稳，避免缓存污染。

5. **LFU 频次规则**

- 计数器非每次访问 + 1，低频易涨、高频难涨；
- 自带**时间衰减**（默认 1 分钟），长期不用的 key 频次自动降低；
- 淘汰时优先删掉访问频率最低的 key。

6. **生产建议**
    
    单实例混存核心数据 + 缓存时，优先用 **volatile-lfu**，安全、命中率更高。
