1. **核心定义**
    
    Redis Pipeline 是批量命令打包机制，将多条命令一次性发送、统一返回结果，核心作用是**减少网络 RTT 往返开销**，提升批量操作性能。
    
2. **核心原理**
    
    客户端缓存多条命令再批量提交，服务端**串行执行**命令，并非并行处理，提速仅来自网络 IO 优化。
    
3. **关键特性**
    

- 无原子性：不属于事务，单条命令失败不影响其他命令执行，无回滚；
- 纯性能优化：和 `MULTI/EXEC` 事务完全不同，二者作用场景分离。

4. **使用限制（重点）**

- 命令不可相互依赖，无法获取上一条执行结果；
- 批量命令不宜过多，会占用服务端、客户端内存与网络缓冲区；
- 不支持阻塞命令、订阅等长连接交互式指令；
- Redis 集群模式下，**无法跨节点 / 跨 slot**使用；
- 命令整体提交，不支持中途取消、部分执行。

5. **适用场景**
    
    适合大批量无依赖的批量增删改查；不适合强原子性、命令联动、集群跨 key 操作的业务。
