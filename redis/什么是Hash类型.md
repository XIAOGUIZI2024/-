1. **你的第一个问题**：Redis 中 hash 是什么？
    
    回答：Hash 是 field-value 嵌套结构，类似 Map / 对象，一个 key 存放多个属性，适合存储用户、商品等实体对象，减少 key 冗余。
    
2. **你的第二个问题**：Redis Hash 底层结构？
    
    回答：Hash 有两种底层编码自动切换：**ziplist（压缩列表）**、**hashtable（字典）**；
    
    满足「字段≤512、单个内容≤64 字节」用 ziplist，超出则转为 hashtable，且不可逆。
    
3. **你的第三个问题**：什么是 ziplist、什么是 hashtable？
    

- ziplist：连续紧凑内存，数据顺序排列，无指针、极度省内存；缺点是遍历查找，性能慢。
- hashtable：数组 + 链表结构，依靠哈希算法快速访问，读写性能高，但内存开销更大。

4. **你的第四个问题**：hashtable 是什么样的数据结构？
    
    回答：底层是 Redis 的 dict 字典，三层结构：
    
    dict（总控制器，维护 2 个哈希表）→ dictht（哈希桶数组）→ dictEntry（键值节点 + 冲突链表）；采用链地址法解决哈希冲突。
    
5. **你的第五个问题**：哈希寻址、渐进式 rehash 是什么？
    

- 哈希寻址：对字段计算哈希值，换算数组下标，直接定位数据位置，实现 O (1) 快速读写。
- 渐进式 rehash：为哈希表扩容 / 缩容的优化方案，避免一次性迁移造成阻塞。

6. **你的第六个问题**：渐进式 rehash 的具体步骤？
    
    ① 触发条件：负载因子过高扩容、过低缩容；
    
    ② 启用备用哈希表 ht [1]，初始化新数组，开启 rehash；
    
    ③ 每次执行命令分批迁移少量数据，逐步搬运；
    
    ④ 迁移期间：新增数据写新表，查询操作先查新表再查旧表；
    
    ⑤ 全部迁移完成，释放旧表内存，新表转正，关闭 rehash。
